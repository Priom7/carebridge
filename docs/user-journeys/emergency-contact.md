# Emergency Contact Journey

Status: Implemented locally in UX-009

1. A caregiver opens **More → Emergency contacts** for the selected care profile.
2. The list shows priority, distance, verification state, and granted capabilities.
3. The caregiver adds or edits identity, phone/WhatsApp, location, availability, and priority.
4. Emergency alerts and basic-profile access default on; medical summary, documents, and location default off.
5. Saving returns to the profile-scoped list. Call and WhatsApp actions use safe demo feedback until platform intents are connected.

The fixture catalog provides 40 fictional contacts across two main care profiles and every verification state.
