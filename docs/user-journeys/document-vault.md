# Document Vault Journey

Status: Implemented locally in UX-010

1. A caregiver selects a care profile and opens **Documents**.
2. The vault shows only that profile's documents and supports title/doctor/tag search, type filtering, and archived visibility.
3. The caregiver chooses a PDF, JPG, PNG, or camera capture and sees upload progress.
4. Unsupported types, files over 20 MB, and camera permission denial produce recoverable guidance.
5. The caregiver records all PRD metadata, links the document to medicine/appointment/condition, and explicitly chooses visibility.
6. The secure viewer obscures content when the application becomes inactive and records view, download, share, archive, retry, and delete activity.

The local demo uses 40 deterministic fictional document records. No real medical files are stored.
