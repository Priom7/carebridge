# Doctor and Appointment Journey

Status: Implemented locally in UX-011

1. A caregiver selects a care profile and opens **More → Appointments**.
2. The profile-scoped list shows doctor, date/time, timezone, location, and lifecycle state.
3. The caregiver can add a doctor with speciality, contact details, address, visiting hours, notes, and linked profile.
4. Scheduling captures reason, questions, reports, attachments, caregiver notes, and timezone. Past dates and another visit within one hour are rejected.
5. Appointment detail presents a non-exportable visit-pack preview using current medicines, questions, reports, and attachments.
6. The caregiver can reschedule, complete the visit, add a summary, and set a follow-up date.
7. Scheduling, updates, rescheduling, and completion add local timeline events for UX-012.

All names and appointments in the demo catalog are fictional.
